Webserv

A C++98 HTTP/1.1 Server Implementation

Project Context

This software was developed as part of the core curriculum at 42 Berlin. The objective of the Webserv project is to understand the intricacies of the Hypertext Transfer Protocol (HTTP) by writing a functional web server from scratch, without the use of external networking libraries.

Technical Specifications

The server is written in C++98 and adheres to the following technical constraints and features:

  • I/O Model: Non-blocking I/O using epoll() for event multiplexing. The server remains responsive under load and handles multiple concurrent client connections.
  • Protocol Support: Implements a subset of RFC 7230-7235 (HTTP/1.1), supporting:
    • Methods: GET, POST, DELETE
    • Transfer-Encoding: chunked request bodies
    • Static file serving and directory listing
    • File uploads
  • CGI Execution: Capability to execute scripts (e.g., PHP, Python) via the Common Gateway Interface, handling environment variables and piping standard input/output.

Configuration

The server behavior is driven by a configuration file inspired by NGINX. Key configurable parameters include:

  • Virtual server setup (host/port binding)
  • Route-specific rules (redirections, root directories, methods)
  • Custom error pages (404, 500, etc.)
  • Client body size limits
